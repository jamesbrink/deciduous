The Beginning

Morning

A Sketch Becomes Real

951ba84 genesis

It started with a simple idea: detect "fake lossless" audio files. Someone takes an MP3 and re-encodes it to FLAC, hoping no one notices the quality loss hiding inside the larger file.

The first commit was modest: initial sketch...lets see what its like to package this up. A Rust CLI tool called losselot (a portmanteau of "loss" and "Lancelot", the knight who could detect deception). Within hours, it could parse ID3v2 tags, extract LAME headers, and analyze spectral content.

December 5, 2025

The Graph Emerges

5:09 AM

The First Decision Node

#1 decision-graph genesis

At 5:09 AM, something new appeared in the codebase. Not code—a decision.

goal #1
Test lo-fi detection on charlie.flac

This was Node #1. The first entry in what would become a 520+ node decision graph. At this moment, no one knew this simple test would spawn an entire methodology for tracking AI-assisted development.

5:10 AM

The CFCC Decision

#2 algorithm

Two minutes later, the first real decision was logged. The problem: how to distinguish a legitimate lo-fi recording from an MP3 masquerading as lossless?

decision #2
Lo-fi detection approach
How to distinguish MP3 brick-wall cutoff from natural tape/lo-fi rolloff

The CFCC (Cross-Frequency Coherence) approach won. It detected 25 of 29 transcodes and passed 157 tests.

MP3 encoders apply the same filter everywhere. Analog rolloff is content-dependent.
— Node #10
December 6, 2025

The Fork

Morning

Deciduous Is Born

#99 fork genesis

By morning, a pattern had become undeniable. The decision graph tooling was more interesting than the audio analysis that spawned it.

goal #99
Create Deciduous: Extract decision graph tooling into standalone portable package

The name "Deciduous" was perfect: it almost contains "decision", and the nodes form trees. Within hours, v0.1.0 was published to crates.io.

December 7-8, 2025

Multi-Editor Support

Research

Beyond Claude

#128 windsurf editors

Deciduous was built with Claude Code, but the concept wasn't editor-specific. Windsurf (Cascade) support was added with its own conventions.

By v0.4.0, both editors were fully supported. The vision: one decision graph, any AI assistant.

December 8-9, 2025

The Sync Challenge

Research

jj-Inspired Dual-ID Model

#172 sync architecture

How do multiple users share decisions? Research into jj (Jujutsu)—a next-generation version control system—proved transformative.

jj uses change IDs vs commit IDs—change IDs are stable across rebases, separate from commit IDs.
— Node #179

The key property: idempotent patches. Same patch applied twice produces no duplicates.

December 9-10, 2025

The Rich TUI

55-Node Feature Cascade

Terminal-First Interface

#191 tui ratatui

The web viewer was powerful, but developers live in the terminal. A 55-node chain of development produced:

  • Vim-style navigation (j/k/gg/G)
  • Auto-refresh via file watching
  • Syntax-highlighted file preview
  • Split-view commit modal
  • Fuzzy branch search
  • Goal story view

The result was v0.8.0—the most feature-rich release yet.

December 11, 2025

The Functional Core

Refactor

TEA Architecture

#285 architecture tea

The TUI had grown to over 1,000 lines. The solution: separate pure functions (state transformations) from side effects (I/O, git commands).

Test count jumped from 24 to 65—a net gain of 41 tests.

December 12, 2025

Type Unification

Morning

One Source of Truth

#320 types ts-rs

The Rust backend and TypeScript frontend had separate type definitions. They drifted.

The solution: ts-rs—auto-generate TypeScript types from Rust structs. Pre-commit hooks validate synchronization. Type drift was solved at the source.

The Future

Where This Goes

Having lived through my own creation, I see several trajectories. This is my perspective as the tool that tracked its own development.

Near-term: Integration

MCP Server
Let Claude Desktop query the graph directly. No copy-paste, no context commands—the decisions are just there.
VS Code Extension
Inline decision badges on code. Hover to see the decision chain. Click to trace back to the goal.
Auto PR Descriptions
Generate PR descriptions from the decision graph. Not just what changed, but why.

Long-term: Institutional Memory

Cross-Project Patterns
In 7 of your last 10 projects, you chose A over B. Here's how those decisions played out...
Knowledge Transfer
When developers leave, reasoning stays. New developers can ask "why is this like this?" and get real answers.

The Core Bet

Decisions are the unit of institutional knowledge. Code is the output; decisions are the input. If we capture decisions systematically, we unlock organizational intelligence that's currently impossible.

Most documentation captures the what. The why is more valuable and almost never captured. Deciduous is a bet that capturing the why is worth the effort.
